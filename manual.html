Logic Gates & Truth Tables
COMBINATIONAL — FUNDAMENTAL GATES
AIM

To study the truth tables and verify the logic operation of fundamental gates: AND, OR, NOT, NAND, NOR, XOR, and XNOR using the VoxLogic simulator.

HOW TO OPERATE
  1. Load Experiment 1 from the sidebar. Gates (AND, OR, NOT, NAND, NOR, XOR, XNOR) appear on the canvas.
  2. Click the Wire tool (W) and connect the INPUT nodes (A, B) to a gate's input ports.
  3. Double-click any INPUT node to toggle its value between 0 and 1.
  4. Observe the output port — green glow = logic 1, dim = logic 0.
  5. Click the ▶▶ Auto Truth Table button in the toolbar to auto-cycle all 4 input combinations.
  6. View the generated truth table in the Truth panel on the right.
  7. Click each gate to see its individual truth table highlighted with the current active row.
KEY POINTS
  • NAND and NOR are Universal Gates — any Boolean function can be implemented using only NAND or only NOR gates.
  • XOR gives Sum bit in adder circuits; AND gives Carry bit.
  • XNOR is the equivalence gate — output is 1 when inputs are equal.
  • NOT gate has only 1 input; all others require 2 or more.
  • Active wire is shown in green (HIGH), inactive in dim blue (LOW).
EXP 02
Universal Gates — NAND & NOR
COMBINATIONAL — GATE REALIZATION
AIM

Realize AND, OR, and NOT functions using only NAND gates. Then realize the same using only NOR gates. Verify outputs match the corresponding basic gate truth tables.

REALIZATION USING NAND

NOT: Tie both inputs of NAND → output = A'

AND: NAND → then invert with NAND-NOT → A·B

OR : Invert A, invert B, then NAND them → A+B (De Morgan)

REALIZATION USING NOR

NOT: Tie both inputs of NOR → output = A'

OR : NOR → invert with NOR-NOT → A+B

AND: Invert A, invert B, then NOR them → A·B (De Morgan)

DE MORGAN'S THEOREM
(A · B)' = A' + B'
(A + B)' = A' · B'

These laws are the mathematical foundation that makes NAND and NOR universal.

HOW TO OPERATE
  1. Load Experiment 2. The canvas shows a NAND-based realization layout.
  2. To build NOT from NAND: drag a NAND gate, right-click → "Add Input Pin" is not needed — instead wire both input ports to the same INPUT node using Wire tool.
  3. For AND from NAND: first NAND gate processes A, B → output feeds a second NAND with both inputs tied together.
  4. Toggle input A and B using double-click. Verify the final output matches the AND truth table.
  5. Click ▶▶ Auto Truth Table to verify all 4 combinations automatically.
  6. Repeat the process with NOR gates for the NOR-based realization.
KEY POINTS
  • A NAND gate with tied inputs behaves as an inverter (NOT gate).
  • Universal gates reduce manufacturing cost — one IC type builds any logic.
  • NAND is more commonly used than NOR in CMOS technology due to speed.
  • Fanout increases with more stages; watch for signal degradation in real circuits.

Half Adder & Half Subtractor
COMBINATIONAL — ARITHMETIC CIRCUITS
AIM

Design and simulate a Half Adder that produces Sum and Carry outputs, and a Half Subtractor that produces Difference and Borrow outputs for 1-bit binary operations.

BOOLEAN EXPRESSIONS

Carry C = A · B

Borrow B = A' · B

HOW TO OPERATE
  1. Load Experiment 3. The canvas shows INPUT nodes A and B, an XOR gate (Sum), and an AND gate (Carry).
  2. Wire INPUT A and B to both the XOR and AND gate inputs using the Wire tool (W key).
  3. Connect the XOR output to the OUTPUT node labeled S (Sum).
  4. Connect the AND output to the OUTPUT node labeled C (Carry).
  5. Double-click A and B to toggle. Verify: when A=1, B=1 → Sum=0, Carry=1 (binary 10 = decimal 2).
  6. For Half Subtractor: replace AND with a NOT-A AND B configuration for the Borrow output.
  7. Use Auto Truth Table to verify all 4 rows simultaneously.
KEY POINTS
  • Called "half" because it cannot accept a carry-in from a previous stage.
  • XOR gate naturally produces the Sum bit; AND gate produces the Carry bit.
  • A=1, B=1 gives Sum=0, Carry=1 representing binary result 10 (decimal 2).
  • Borrow is 1 only when A=0, B=1 (subtracting 1 from 0 requires borrowing).
  • Circuit uses exactly 2 gates — minimum possible for 1-bit addition.

Full Adder & Full Subtractor
COMBINATIONAL — CASCADEABLE ARITHMETIC
AIM

Implement a Full Adder with carry-in input enabling multi-bit cascading, and a Full Subtractor with borrow-in input. Verify all 8 input combinations.

Sum S = A ⊕ B ⊕ Cin

HOW TO OPERATE
  1. Load Experiment 4. Three INPUT gates (A, B, Cin) are present.
  2. Stage 1: Wire A and B into XOR1 (intermediate P) and AND1 (intermediate Q).
  3. Stage 2: Wire P and Cin into XOR2 → this gives final Sum output S.
  4. Wire P and Cin into AND2 (intermediate R). Wire Q and R into an OR gate → this gives Cout.
  5. Toggle A, B, Cin via double-click and verify all 8 combinations using Auto Truth Table.
  6. Critical test: A=1, B=1, Cin=1 → Sum=1, Cout=1 (represents decimal 3 = binary 11).
KEY POINTS
  • Full Adder = two Half Adders + an OR gate for the carry.
  • Carry-out (Cout) of one stage becomes Carry-in (Cin) of the next — enabling cascading.
  • Standard building block of all multi-bit binary adders.
  • Uses 5 gates total: 2 XOR, 2 AND, 1 OR.
EXP 05
4-bit Ripple Carry Adder
COMBINATIONAL — MULTI-BIT ARITHMETIC
AIM

Chain four Full Adders to add two 4-bit numbers A[3:0] and B[3:0] producing a 5-bit output (Cout, S3, S2, S1, S0).

STRUCTURE

FA0: A0+B0+0 → S0, C0
FA1: A1+B1+C0 → S1, C1
FA2: A2+B2+C1 → S2, C2
FA3: A3+B3+C2 → S3, Cout

HOW TO OPERATE
  1. Load Experiment 5. Set all input bits A3–A0 and B3–B0 by double-clicking their INPUT nodes.
  2. The Cin of FA0 is permanently wired to 0.
  3. Cout of each FA feeds the Cin of the next FA to the left.
  4. Example: Set A=0111 (7) and B=0001 (1). Expected output: S=1000 (8), Cout=0.
  5. Try A=1111 (15) + B=0001 (1) = 10000 → Cout=1, S=0000 (overflow).
  6. Step through the simulation using the STEP button to observe carry propagation.
KEY POINTS
  • Critical path delay = 4 × (Full Adder delay) due to sequential carry propagation.
  • Carry Look-Ahead Adder (CLA) eliminates this by computing all carries simultaneously.
  • IC 74LS83 is a 4-bit binary adder used in real hardware.
  • Maximum sum for 4-bit + 4-bit is 30 (1111 + 1111 = 11110).
EXP 06
BCD Adder Design
COMBINATIONAL — DECIMAL ARITHMETIC
AIM

Add two BCD digits (0–9) and produce a valid BCD result. Apply 0110 correction when the binary sum exceeds 9 or a carry is generated.

CORRECTION LOGIC
Correct = C + (S3·S2) + (S3·S1)

When Correct=1, add 0110 (6) to the binary sum to skip invalid codes 1010–1111.

HOW TO OPERATE
  1. Load Experiment 6. Enter BCD digits A (A3–A0) and B (B3–B0) using INPUT nodes.
  2. Stage 1 binary adder produces intermediate sum S[3:0] and carry C.
  3. Observe the correction detect logic — if sum > 9, green wire on the correction gate.
  4. Stage 2 adder adds 0110 when correction is needed.
  5. Test: BCD 5 (0101) + BCD 7 (0111) = binary 12 (1100). Correction adds 0110 → result is 10010 = BCD 1 and 2 = decimal 12.
KEY POINTS
  • BCD uses 4 bits per decimal digit; codes 1010–1111 are invalid and never occur in output.
  • Correction value 0110 (6) skips the 6 invalid codes (10–15).
  • Used in calculators, ATMs, and financial computing where decimal precision is mandatory.
  • Two 4-bit adder stages are required — first for binary sum, second for correction.

HOW TO OPERATE
  1. Load Experiment 7. Input nodes B3–B0 represent the 4-bit binary input.
  2. XOR gates implement the Gray conversion: G3=B3 (BUF), G2=B3⊕B2, G1=B2⊕B1, G0=B1⊕B0.
  3. Toggle inputs to binary values (e.g., 0110 = decimal 6) and read Gray output.
  4. Verify that consecutive Gray codes differ by exactly 1 bit — this is the defining property.
  5. For BCD to Excess-3: add 0011 (3) to each BCD digit — verify using the correction adder structure.
KEY POINTS
  • Gray code changes only 1 bit between consecutive values — prevents glitches in mechanical encoders.
  • Used extensively in Karnaugh maps for logic minimization.
  • Excess-3 is self-complementing: 9's complement of any BCD digit is its bit-inversion in Excess-3.
  • Gray code conversion uses only XOR gates — simple and fast.
EXP 08
Encoders & Decoders
COMBINATIONAL — DATA ROUTING
3-TO-8 DECODER

Inputs: A, B, C (3 bits). Outputs: D0–D7 (8 lines). Exactly one output is active for each input combination. Each output = minterm of inputs.

D0 = A'B'C'   D1 = A'B'C
D2 = A'BC'   D3 = A'BC
D4 = AB'C'   D5 = AB'C
D6 = ABC'    D7 = ABC
PRIORITY ENCODER (8-TO-3)

Converts active input line to 3-bit binary output. If multiple inputs active, highest-numbered input takes priority. Essential for interrupt controllers.

HOW TO OPERATE
  1. Load Experiment 8. Three INPUT nodes A, B, C feed the decoder NAND gates.
  2. Toggle A, B, C to binary value 101 (decimal 5). Verify only D5 output glows green.
  3. For each of the 8 combinations (000–111), confirm exactly one output line is active.
  4. Switch to encoder mode: activate only one input line and verify the 3-bit binary output.
  5. For priority encoder testing: activate multiple inputs simultaneously and confirm highest takes priority.
KEY POINTS
  • A 3-to-8 decoder has 3 inputs and 2³ = 8 outputs.
  • Active-LOW decoders use NAND gates; active-HIGH use AND gates.
  • Enable pin allows cascading: two 3-to-8 decoders → one 4-to-16 decoder.
  • Used in memory address decoding and 7-segment display drivers.

Select lines S1, S0 choose which data input (I0–I3) passes to output Y.

HOW TO OPERATE
  1. Load Experiment 9. Six INPUT nodes: I0–I3 (data) and S0, S1 (select).
  2. Set S1=0, S0=0 to select I0. Toggle I0 and observe output Y following I0's value.
  3. Set S1=1, S0=1 to select I3. Toggle I3 and confirm Y follows I3.
  4. To implement a function: tie data inputs to 0 or 1 according to the truth table minterm positions.
  5. For DEMUX: set data input D and change select lines to route D to different outputs.
KEY POINTS
  • MUX can implement any Boolean function of n+1 variables using a 2ⁿ:1 MUX.
  • DEMUX is essentially a decoder with a data input — same hardware, different use.
  • IC 74LS153 is a dual 4:1 MUX; IC 74LS139 is a dual 2:4 DEMUX/decoder.
  • Used in time-division multiplexing to share one wire among multiple signals.
EXP 10
SR Flip-Flop (NAND)
SEQUENTIAL — BISTABLE MEMORY ELEMENT
STATE TABLE
S̄R̄QQ'State
11No changeHOLD
0110SET
1001RESET
0011FORBIDDEN
CIRCUIT DESCRIPTION

Two cross-coupled NAND gates. Gate 1 takes S̄ and Q' as inputs → outputs Q. Gate 2 takes R̄ and Q as inputs → outputs Q'. Active-LOW inputs (0 activates function).

HOW TO OPERATE
  1. Load Experiment 10. Two INPUT nodes S and R, two NAND gates cross-coupled, two OUTPUT nodes Q and Q'.
  2. SET operation: Set S=0, R=1. Verify Q goes to 1 and Q' goes to 0.
  3. Release S back to 1 (S=1, R=1). Verify Q remains 1 — this demonstrates memory.
  4. RESET operation: Set R=0, S=1. Verify Q goes to 0 and Q' goes to 1.
  5. Release R back to 1. Verify Q remains 0 — memory retained again.
  6. Observe the waveform in the Wave panel showing Q holding its state between toggles.
  7. Avoid: Setting both S=0 and R=0 simultaneously — forbidden state.
KEY POINTS
  • First sequential circuit: output depends on both current inputs AND previous state.
  • SR NAND latch uses active-LOW inputs: S̄=0 sets, R̄=0 resets.
  • Forbidden state (S=0, R=0) forces Q=Q'=1, violating complementary property.
  • Used to debounce mechanical switches — eliminates contact bounce noise.
  • SR latch is the building block of all other flip-flops.
EXP 11
JK & D Flip-Flops
SEQUENTIAL — CLOCKED MEMORY ELEMENTS
JK FLIP-FLOP STATE TABLE
JKQn+1Operation
00QnNo change
010RESET

D FLIP-FLOP
Qn+1 = D (on clock edge)

D flip-flop samples and latches data input D on each rising clock edge. Eliminates forbidden state by internally connecting D to J and D' to K.

HOW TO OPERATE
  1. Load Experiment 11. INPUT nodes: J, K, CLK for JK flip-flop.
  2. Set J=1, K=0, then toggle CLK (0→1). Observe Q goes to 1 (SET operation).
  3. Set J=0, K=1, toggle CLK. Observe Q goes to 0 (RESET operation).
  4. Set J=1, K=1, toggle CLK repeatedly. Observe Q toggling with each clock pulse.
  5. For D flip-flop: connect D input and clock it. Q should follow D after each clock edge.
  6. Use the STEP button to advance clock one tick at a time and observe state changes.
  7. View the waveform in the Wave tab to see CLK, Q, Q' timing relationships.
KEY POINTS
  • JK flip-flop resolves the SR forbidden state — J=K=1 gives toggle instead of invalid output.
  • Race-around condition occurs in level-triggered JK when J=K=1 and clock stays HIGH.
  • D flip-flop is the most common storage element in synchronous digital design.
  • T flip-flop (toggle) is a JK with J=K=1 permanently — used in frequency dividers.
  • One D flip-flop = one bit of register storage in a CPU.

SHIFT REGISTER (SIPO)

4 D flip-flops in series. Serial data enters FF1. On each clock, each FF captures output of previous FF.

After 4 clocks with input 1,0,1,1:
Q3 Q2 Q1 Q0 = 1 0 1 1

SIPO = Serial-In Parallel-Out
SISO = Serial-In Serial-Out
PIPO = Parallel-In Parallel-Out

HOW TO OPERATE
  1. Load Experiment 12. CLK and D-in INPUT nodes are present.
  2. For counter: press STEP repeatedly. Observe Q2, Q1, Q0 incrementing in binary from 000 to 111.
  3. After step 8, verify counter wraps back to 000 (modulo-8 behavior).
  4. Switch to Auto-Sim for automatic clocking. Observe waveform in Wave tab.
  5. For shift register: set D-in to a data bit (0 or 1), press STEP 4 times loading 4 bits.
  6. After 4 steps, read Q3–Q0 as the parallel output of the 4 loaded bits.
  7. Try loading the pattern 1011 serially and verify parallel output equals 1011 after 4 clock steps.
KEY POINTS
  • Ripple counter is asynchronous — flip-flops clock at different times causing glitches.
  • Synchronous counters (all FFs clocked simultaneously) are glitch-free and faster.
  • 3-bit counter counts modulo-8; add NAND reset detection for modulo-N counter.
  • Shift registers convert between serial and parallel data formats.
  • IC 74HC194 is a universal shift register with all four modes (SISO, SIPO, PISO, PIPO).
  • Counters form the basis of program counters, timers, and frequency dividers in CPUs.
